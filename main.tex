\chapter*{Nota para los autores}
\addcontentsline{toc}{chapter}{Nota para los autores}

Este documento corresponde al \textbf{informe final del sistema}. Su propósito es consolidar de forma organizada los aportes técnicos de los diferentes participantes.

Cada capítulo debe documentar no solo el resultado final, sino también:

\begin{itemize}
    \item El propósito del componente o tema.
    \item Su relación con el sistema general.
    \item Las decisiones técnicas tomadas.
    \item Los problemas encontrados.
    \item Los ajustes realizados.
    \item Los criterios usados para decidir dichos ajustes.
    \item Las pruebas o evidencias disponibles.
    \item Los aprendizajes derivados de la práctica.
    \item Los enlaces reales a repositorios de software, cuando aplique.
\end{itemize}

Si se menciona software, no se debe incluir el código fuente completo en este informe. En su lugar, se debe describir su función, entradas, salidas, integración y agregar el enlace al repositorio real correspondiente.

Cuando no exista un repositorio publicado, se debe escribir explícitamente: \textit{Repositorio pendiente de publicación}. No se deben inventar enlaces.
