\section{An\'alisis de requerimientos, restricciones y uso}

Esta secci\'on debe consolidar los requerimientos funcionales y no funcionales del sistema,
as\'i como las restricciones de hardware, software, conectividad, presupuesto, tiempo o
entorno de despliegue que condicionan el dise\~no.

El placeholder actual deja definido el espacio donde luego podr\'an incluirse tablas,
listas o descripciones de casos de uso que permitan justificar decisiones de arquitectura
y priorizaci\'on t\'ecnica.

\subsection*{Puntos sugeridos para completar}
\begin{itemize}[leftmargin=*]
  \item Requerimientos funcionales del sistema.
  \item Requerimientos no funcionales: latencia, disponibilidad, seguridad, escalabilidad.
  \item Restricciones de adquisici\'on, procesamiento y transmisi\'on de datos.
  \item Limitaciones del entorno operativo o experimental.
  \item Casos de uso o escenarios de operaci\'on.
\end{itemize}
