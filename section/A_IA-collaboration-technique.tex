\chapter{Técnica de desarrollo en grupo usando agentes IA}
\label{annex:ia_collaboration}

\section{Propósito del capítulo}
Este anexo tiene como objetivo documentar la metodología de trabajo colaborativo y la infraestructura técnica utilizada por el equipo de desarrollo al integrarse con agentes de Inteligencia Artificial (IA). Se busca proporcionar una guía clara sobre la instalación, configuración y mejores prácticas de \textit{prompting} para asegurar que el uso de estas herramientas sea trazable, repetible y de alta calidad dentro del ciclo de vida del proyecto.

\section{Contexto dentro del sistema general}
En el desarrollo de sistemas complejos como el de adquisición SDR y plataformas \textit{cloud}, el uso de agentes IA (como Gemini CLI y Copilot CLI) permite acelerar la automatización de flujos de trabajo, la revisión de código y la generación de documentación técnica. Esta técnica no reemplaza la supervisión humana, sino que actúa como un multiplicador de capacidades, integrando razonamiento multi-agente para la toma de decisiones técnicas y el análisis de contexto.

\section{Desarrollo técnico: Ecosistema de Agentes CLI}
El equipo utiliza una combinación de herramientas de terminal que permiten una interacción directa con modelos de lenguaje avanzados.

\subsection{Instalación y Configuración}
A continuación se detallan los comandos principales para habilitar el entorno de agentes:

\begin{itemize}
    \item \textbf{GitHub Copilot CLI}: Utilizado principalmente para asistencia en codificación y automatización de tareas de repositorio.
    \begin{itemize}
        \item Instalación: \texttt{npm install -g @github/copilot}
        \item Autenticación: \texttt{copilot /login}
    \end{itemize}
    \item \textbf{Gemini CLI}: Utilizado para tareas multimodales, gestión de habilidades (\textit{skills}) y análisis de arquitectura.
    \begin{itemize}
        \item Instalación: \texttt{npm install -g @google/gemini-cli}
        \item Inicio: \texttt{gemini}
    \end{itemize}
\end{itemize}

\subsection{Arquitectura Multi-Agente}
El sistema se diseña bajo un esquema de roles especializados que colaboran para resolver tareas complejas:

\begin{itemize}
    \item \textbf{Agente Coordinador}: Asigna roles y selecciona el modelo más apto.
    \item \textbf{Agente de Interpretación}: Analiza el contexto de la petición.
    \item \textbf{Agente de Backend}: Gestiona el almacenamiento y la comunicación con APIs.
    \item \textbf{Agente de Procesamiento}: Maneja tareas específicas de señales (audio/espectro).
\end{itemize}

\section{Decisiones de diseño: Metodología de Prompting}
Para minimizar alucinaciones y maximizar la coherencia de las respuestas, se ha adoptado una estructura obligatoria para cada interacción con los agentes, basada en el modelo \textbf{ROLE \(\rightarrow\) OBJECTIVE \(\rightarrow\) CONTEXT \(\rightarrow\) REASONING}.

\begin{table}[H]
\centering
\caption{Componentes de un Prompt de alta calidad.}
\label{tab:prompt_components}
\begin{tabularx}{\textwidth}{>{\bfseries}l X}
\toprule
Componente & Descripción \\
\midrule
Role & Define quién debe responder (e.g., "Senior Software Developer"). \\
Objective & Define qué se quiere lograr de forma clara y concisa. \\
Context & Proporciona archivos relevantes, restricciones y entorno. \\
Reasoning & Solicita al agente explicar su lógica antes de entregar el resultado. \\
Output Format & Especifica el formato deseado (JSON, LaTeX, C code, etc.). \\
\bottomrule
\end{tabularx}
\end{table}

\section{Aprendizajes y transferencia de conocimiento}
\begin{itemize}
    \item \textbf{Criterio de Realidad}: La IA es un asistente, no un ejecutor autónomo sin supervisión. La validación humana es mandatoria.
    \item \textbf{Modulariad}: Es preferible pedir tareas pequeñas y específicas que una solución completa y compleja en un solo prompt.
    \item \textbf{Documentación}: Mantener archivos \texttt{GEMINI.md} o \texttt{AGENTS\_CLI.md} en la raíz del proyecto ayuda a los agentes a entender rápidamente las convenciones del equipo.
\end{itemize}

\section{Recomendaciones específicas}
Se recomienda la rotación de roles entre los desarrolladores para familiarizarse con las capacidades de diferentes \textit{backends} (GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro) según la naturaleza de la tarea.
