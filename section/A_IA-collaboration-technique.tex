\chapter{T\'ecnica de desarrollo en grupo usando agentes IA}
\label{annex:ia_collaboration}

\section{Prop\'osito del cap\'itulo}
Este anexo tiene como objetivo documentar la metodolog\'ia de trabajo colaborativo y la infraestructura t\'ecnica utilizada por el equipo de desarrollo al integrarse con agentes de Inteligencia Artificial (IA). Se busca proporcionar una gu\'ia clara sobre la instalaci\'on, configuraci\'on y mejores pr\'acticas de \textit{prompting} para asegurar que el uso de estas herramientas sea trazable, repetible y de alta calidad dentro del ciclo de vida del proyecto.

\section{Contexto dentro del sistema general}
En el desarrollo de sistemas complejos como el de adquisici\'on SDR y plataformas \textit{cloud}, el uso de agentes IA (como Gemini CLI y Copilot CLI) permite acelerar la automatizaci\'on de flujos de trabajo, la revisi\'on de c\'odigo y la generaci\'on de documentaci\'on t\'ecnica. Esta t\'ecnica no reemplaza la supervisi\'on humana, sino que act\'ua como un multiplicador de capacidades, integrando razonamiento multi-agente para la toma de decisiones t\'ecnicas y el an\'alisis de contexto.

\section{Desarrollo t\'ecnico: Ecosistema de Agentes CLI}
El equipo utiliza una combinaci\'on de herramientas de terminal que permiten una interacci\'on directa con modelos de lenguaje avanzados.

\subsection{Instalaci\'on y Configuraci\'on}
A continuaci\'on se detallan los comandos principales para habilitar el entorno de agentes:

\begin{itemize}
    \item \textbf{GitHub Copilot CLI}: Utilizado principalmente para asistencia en codificaci\'on y automatizaci\'on de tareas de repositorio.
    \begin{itemize}
        \item Instalaci\'on: \texttt{npm install -g @github/copilot}
        \item Autenticaci\'on: \texttt{copilot /login}
    \end{itemize}
    \item \textbf{Gemini CLI}: Utilizado para tareas multimodales, gesti\'on de habilidades (\textit{skills}) y an\'alisis de arquitectura.
    \begin{itemize}
        \item Instalaci\'on: \texttt{npm install -g @google/gemini-cli}
        \item Inicio: \texttt{gemini}
    \end{itemize}
\end{itemize}

\subsection{Arquitectura Multi-Agente}
El sistema se dise\~na bajo un esquema de roles especializados que colaboran para resolver tareas complejas:

\begin{itemize}
    \item \textbf{Agente Coordinador}: Asigna roles y selecciona el modelo m\'as apto.
    \item \textbf{Agente de Interpretaci\'on}: Analiza el contexto de la petici\'on.
    \item \textbf{Agente de Backend}: Gestiona el almacenamiento y la comunicaci\'on con APIs.
    \item \textbf{Agente de Procesamiento}: Maneja tareas espec\'ificas de se\~nales (audio/espectro).
\end{itemize}

\section{Decisiones de dise\~no: Metodolog\'ia de Prompting}
Para minimizar alucinaciones y maximizar la coherencia de las respuestas, se ha adoptado una estructura obligatoria para cada interacci\'on con los agentes, basada en el modelo \textbf{ROLE \(\rightarrow\) OBJECTIVE \(\rightarrow\) CONTEXT \(\rightarrow\) REASONING}.

\centering

\toprule
Componente & Descripci\'on \\
\midrule
Role & Define qui\'en debe responder (e.g., "Senior Software Developer"). \\
Objective & Define qu\'e se quiere lograr de forma clara y concisa. \\
Context & Proporciona archivos relevantes, restricciones y entorno. \\
Reasoning & Solicita al agente explicar su l\'ogica antes de entregar el resultado. \\

\section{Validaci\'on y Evidencias}
La metodolog\'ia se valida mediante la revisi\'on humana de cada entrega. El flujo de trabajo colaborativo asegura que ning\'un cambio se integre sin pasar por un ciclo de validaci\'on cruzada entre agentes y desarrolladores.

\section{Aprendizajes y transferencia de conocimiento}
\begin{itemize}
    \item \textbf{Criterio de Realidad}: La IA es un asistente, no un ejecutor aut\'onomo sin supervisi\'on. La validaci\'on humana es mandatoria.
    \item \textbf{Modulariad}: Es preferible pedir tareas peque\~nas y espec\'ificas que una soluci\'on completa y compleja en un solo prompt.
    \item \textbf{Documentaci\'on}: Mantener archivos \texttt{GEMINI.md} o \texttt{AGENTS\_CLI.md} en la ra\'iz del proyecto ayuda a los agentes a entender r\'apidamente las convenciones del equipo.
\end{itemize}

\section{Recomendaciones espec\'ificas}
Se recomienda la rotaci\'on de roles entre los desarrolladores para familiarizarse con las capacidades de diferentes \textit{backends} (GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro) seg\'un la naturaleza de la tarea.
